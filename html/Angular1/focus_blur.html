directive封装输入框获取焦点和失去焦点

作者：吴杨杨 返回首页

时间：2017-02-13

代码如下：

angular.module('artiteckApp')
   .directive('pzFocusBlur', function() {
      return {
         restrict: 'A',
         scope: false,
         link: function(scope, elem, attrs, ngModelCtr) {
            // 失去焦点时
            elem.on('blur', function() {
               scope.$apply(function() {
                  scope.pzBtn = true;
               });
            });
            
            // 获得焦点时
            elem.on('focus', function() {
               scope.$apply(function() {
                  scope.pzBtn = false;
               });
            });
         }
      };
   });

如果要修改父作用域的值，必须加上如下代码

  • scope: false

scope: false意思是继承父作用域，这样我们创建的指令和父作用域（其实是同一个作用域）共享同一个model模型，所以在指令中修改pzBtn数据，它会反映到父作用域的模型中。

如果我们将scope设置为true,我们就会创建一个新的作用域，这个作用域继承了我们的父作用域，但是它和父作用域不是同一个作用域。

并且，还得加一个$apply方法，来更新父作用域数据

scope.$apply(function() {
   scope.pzBtn = false;
});

这样，父作用于的pzBtn值才会发生改变
