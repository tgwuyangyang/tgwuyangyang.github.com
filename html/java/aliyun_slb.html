阿里云服务器均衡负载SLB

作者：吴杨杨 返回首页

时间：2016-08-18

最近，公司因为服务器压力以及对未来规划等原因，让我研究解决方法。查了几天资料，终于找到一些资料，分享给那些和我一样正在慢慢摸索的人。

阿里云SLB, 负载均衡（Server Load Balancer）是对多台云服务器进行流量分发的负载均衡服务。

用户云服务器必须2台＋, 直接在阿里云官网申请开通SLB即可。

开启负载均衡的好处，两台服务器分担网站数据请求实现均衡，相当于组建一个小型的服务器群集。

阿里的负载均衡SLB是这样运作的。现有A服务器，A安装配置有tomcat以及数据库，A现在是可以正常使用的。新增一台B服务器，B服务器复制A服务器tomcat，B服务器的数据库调用要设置成A，这样保证调用相同数据。但需要配置均衡AB的权重，实现更好的分担和均衡效果。

或者再新增一台C服务器，C服务器配置数据库，A和B安装tomcat，现在A和B的tomcat配置完全一样，数据库调用都指向C。

在网站高峰期时查看服务器AB资源使用率，根据使用率的高低分别设置不同的权重级别。

开启负载均衡的好处，两台服务器分担网站数据请求实现均衡，相当于组建一个小型的服务器群集。

SLB会通过健康检查功能，实时监控后端ECS的运行状态。如其中1台或多台ECS异常无法提供服务，SLB会自动屏蔽异常ECS，保证整体应用系统的可用。

使用SLB服务，访问者无法获取到整体应用系统后端ECS的真是IP。

but

SLB更侧重于提升网站的可用性，提升负载能力倒是次要了，当你网站需要多台服务器进行热备的时候SLB的重要性不言而喻。

于瞬间的并发请求，SLB无用武之地。SLB只转发请求，并不拆分请求。

负载均衡技术一般实现两个目的：1.将过多请求分散到不同的机器上 2.某一节点出现问题的时候将其移除，保证集群的可用性 。
